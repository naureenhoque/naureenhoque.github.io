# Academic GitHub Pages Website

A responsive, dependency-free academic personal website for an Assistant Professor working across:

- Artificial intelligence
- Cybersecurity
- Wireless systems
- Post-quantum security

## Files

- `index.html` — all website content
- `styles.css` — layout and visual design
- `script.js` — mobile navigation and current year
- `README.md` — these instructions

## Personalize the website

Open `index.html` and replace every placeholder, especially:

- `Dr. Your Name`
- `YN`
- `[University Name]`
- `[Department Name]`
- `your.email@university.edu`
- `[Building and Room]`
- Biography and education
- Course codes, terms, descriptions, and links
- Publication titles, authors, venues, years, and links
- Google Scholar, ORCID, GitHub, and LinkedIn URLs

Search for `EDIT:` and `[` to find the main placeholders quickly.

## Add your photograph

1. Add a photograph named `profile.jpg` to the repository root.
2. In `index.html`, replace the `portrait-placeholder` block with the commented
   `<img class="profile-photo" ...>` line.

## Add your CV

Add your CV to the repository root and name it `cv.pdf`. The existing CV button
will then work automatically.

## Publish with GitHub Pages

1. Create a GitHub repository named exactly `YOUR-USERNAME.github.io`.
2. Upload `index.html`, `styles.css`, `script.js`, and any assets.
3. Open **Settings → Pages**.
4. Under **Build and deployment**, choose **Deploy from a branch**.
5. Select the `main` branch and `/ (root)`, then save.
6. Visit `https://YOUR-USERNAME.github.io`.

## Updating the website

Edit the files in GitHub and commit your changes. GitHub Pages republishes the
site after the commit is processed.
